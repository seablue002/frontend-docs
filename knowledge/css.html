    # 1. css盒模型 初级

    # 标准模式与怪异模式(标准盒模型与IE盒模型)

    由于历史原因，早期W3C未完全制定下来的时候，浏览器生产产商之间都各自有各自的实现网页渲染、展示标准，在W3C标准制定后总的来说归为两大类渲染展示模式。

    1.标准模式standard mode

    2.混杂模式（怪异模式）quirks mode

    模式名称 采用情形

    混杂模式（怪异模式）quirks mode 没有doctype声明的采用quirks mode解析

    标准模式standard mode 对于有doctype的大多数采用standard mode

    # 二者区别

    • 在standard mode中: width用作内容content宽度

    • 在quirks mode中： width用作元素的宽度，内容content宽度 = width - (padding-left + padding-right + border-left-width + border-right-width)

    # 如何触发这两种模式

    • 浏览器根据doctype是否存在以及声明使用哪种DTD来呈现。
    • 如果XHTML文档包含形式完整的doctype，则以一般标准模式呈现。
    • 对于html4.0.1文档，包含严格DTD的doctype以标准模式呈现页面，包含过渡DTD和URI的doctype以标准模式呈现页面，但是有过渡DTD没有URI则以混杂模式呈现。Doctype不存在或者行事不正确则以混杂模式呈现
    • CSS3新增的属性box-sizing，能够用来修改盒模型，可选值为：
    • content-box：默认值，让元素维持W3C的标准盒模型
    • border-box：让元素维持IE传统（混杂模式）的盒模型

    # 2. 实现移动端Retina屏幕1px边框效果 初级

    一般在pc电脑或者低清晰度屏幕上 1css px = 1物理 px像素， 所以我们在css中定义 border: 1px solid #ccc; 就表示使用1css px, 占据屏幕上1像素点

    但是当移动端出现高清晰度屏幕（视网膜）retina屏，屏幕有了更多的物理像素， @x2 倍屏幕有 @x1屏幕两倍物理像素时，我们如果依然使用 1 css px 定义边框时，边框看起来比1像素要宽，因为这时候由于高清屏的1 css px像素单位对应2倍 物理像素

    如果我们需要在高清屏显示1px效果，需要使用 css3 transform: scale来实现

    # 3. BFC 中级

    （Block Formatting Context，BFC） 块级格式化上下文，它是页面中的一块渲染区域，并且有一套属于自己的渲染规则，它决定了元素如何对齐内容进行布局，以及与其他元素的关系和相互作用。 当涉及到可视化布局的时候，BFC提供了一个环境，HTML元素在这个环境中按照一定规则进行布局。 BFC是一个独立的布局环境，BFC内部的元素布局与外部互不影响。

    # 1、BFC的布局规则

    1.内部的Box会在垂直方向一个接着一个地放置。

    2.Box垂直方向上的距离由margin决定。属于同一个BFC的两个相邻的Box的margin会发生重叠。

    3.每个盒子的左外边框紧挨着包含块的左边框，即使浮动元素也是如此。

    4.BFC的区域不会与float box重叠。

    5.BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素，反之亦然。

    6.计算BFC的高度时，浮动子元素也参与计算。

    1、2、3条非BFC的块级元素也具有，4、5、6仅BFC具有

    1、创建（触发）BFC的常用方式 根元素（<html>）

    浮动元素（float 值不为 none）

    绝对定位元素（position 值为 absolute 或 fixed）

    行内块元素（display 值为 inline-block）

    表格单元格（display 值为 table-cell，HTML表格单元格默认值）

    表格标题（display 值为 table-caption，HTML表格标题默认值）

    匿名表格单元格元素（display 值为 table、table-row、 table-row-group、table-header-group、table-footer-group（分别是 HTML table、tr、tbody、thead、tfoot 的默认值）或 inline-table）

    overflow 值不为 visible、clip 的块元素

    display 值为 flow-root 的元素

    contain 值为 layout、content 或 paint 的元素

    弹性元素（display 值为 flex 或 inline-flex 元素的直接子元素），如果它们本身既不是flex、grid也不是table容器

    网格元素（display 值为 grid 或 inline-grid 元素的直接子元素），如果它们本身既不是flex、grid也不是table容器

    多列容器（column-count 或 column-width (en-US) 值不为 auto，包括column-count 为 1）

    column-span 值为 all 的元素始终会创建一个新的 BFC，即使该元素没有包裹在一个多列容器中

    # 3、BFC的运用场景

    1.解决浮动元素令父元素高度坍塌的问题

    方法：给父元素开启BFC

    原理：计算BFC的高度时，浮动子元素也参与计算

    2.非浮动元素被浮动元素覆盖

    方法：给非浮动元素开启BFC

    原理：BFC的区域不会与float box重叠

    3.两栏自适应布局

    方法：给固定栏设置固定宽度和float，给不固定栏开启BFC

    原理：BFC的区域不会与float box重叠

    4.外边距垂直方向重合的问题

    方法：给上box或者下box任意一个包裹新的box并开启BFC

    原理：属于同一个BFC的两个相邻的Box的margin会发生重叠

    BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素，反之亦然
